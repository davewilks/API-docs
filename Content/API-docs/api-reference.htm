Green Defenestrator API Reference

The Green Defenestrator Audit & Forensics Logs API provides read-only access to forensics and audit data that Green Defenestrator nodes collect. These APIs provide the ability to integrate Green Defenestrator with external applications, such as SIEM solutions like Microsoft Sentinel. The Audit and Forensics logs API includes two resources: logs/payload and logs/full-payload. The primary difference between the two is logs/payload does not return Personally Identifiable Information (PII), while logs/full-payload does.

If a user cancels opening a large file, Green Defenestrator does not log the decryption attempt.

full-payload

The full-payload endpoint contains all the events associated with a file, including protect, open, and audit events. It includes PII and sensitive information, like names and email addresses. If you do not want to return PII, use the payload resource instead.

Full-Payload Attributes

The following table lists the attributes of this object.

What's included in the JSON object depends on the permissions assigned to the API user. Work with your administrator to ensure that you have the right permissions.

Attribute

Description

pagination

object

Provides pagination details. You can pass page and pageSize query parameters to control the amount of data you receive.

data

array

Contains the attributes and values of the object retrieved.

logID

string

 The unique identifier of the log record in Green Defenestrator.

userLastNameFirstName

string

The full name of the user associated with the logged record, formatted as LastName, FirstName.

userLastNameFirstName is PII.

userEmailAddress

string

The email address of the user associated with the log record.

userEmailAddress is PII.

currentOwnerLastNameFirstName

string

The first and last name of the owner of the Green Defenestrator-protected file.

currentOwnerLastNameFirstName is PII.

currentOwnerEmailAddress

string

The first and last name of the owner of the Green Defenestrator-protected file.

currentOwnerEmailAddress is PII.

userId

string

The unique identifier of the user associated with the log record. This is a Green Defenestrator-generated internal user identifier unique to a Green Defenestrator node.

payloadId

string

The unique identifier associated with the Green Defenestrator-protected file. This is a Green Defenestrator-generated internal payload identifier unique to a Green Defenestrator node.

payloadName

string

The name of the Green Defenestrator-protected file.

currentPayloadOwnerId

string

The unique identifier of the owner of the Green Defenestrator-protected file.

actionAttempted

string

The type of logged action (e.g., Decrypt, Audit, Encrypt, or EncryptionRequest). The actionAttempted always includes a related result and resultReason. See Audit & Forensics Logs Actions, Results, and Reasons for details.

result

string

How Green Defenestrator responded to the actionAttempted. Possible values include AccessGranted, AccessDenied, AccessWindow, RevokeAccess, AuthorisedGroupAccess, AuthorisedUserAccess, and AuthorChanged. See Audit & Forensics Logs Actions, Results, and Reasons for details.

resultReason

string

Why Green Defenestrator returned the result of the actionAttempted. Possible values include Changed, Created, Success, TimeEmbargoFailed, UserPayloadNoAccess, or GeoValidationFailed. See Audit & Forensics Logs Actions, Results, and Reasons for details.

logTimestamp

string

The UTC date and time of the log record, formatted as an ISO 8601 date and time: YYYY-MM-DDTHH:MM:SS.ss.

userNetwork

string

The network information used to identify geolocation information for the user's network, including IP address, network name, network ID, domain name, device type, machine name, MAC address, UUID, service provider, latitude and longitude, and address.

Access to geolocation information requires elevated privileges.

oId

string

The identity provider's unique identifier for a user authenticating with Green Defenestrator using credentials from an external IdP. The value is null for user's authenticating with credentials managed by their organization.

oIdProviderName

string

The name of the identity provider associated with the oId.

Retrieve Full-Payload

Use a GET /logs/full-payload request to retrieve the list of logged events, based on the authentication used to make the request.

Copy 
curl --location 'https://<Custom API URL>/api/logs/full-payload' \
--header 'Authorization: Bearer <Your Token>'

Parameters

The following table shows the query parameters available to tailor and filter the response for this resource.

Parameter

Description

actionAttempted

string

Filter by Decrypt, Audit, or Encrypt.

currentPayloadOwnerId

string

Filter by the unique identifier of the owner of the Green Defenestrator-protected file.

endTime

string

Filter by the concluding time and date of any file contained in the query to retrieve records modified before a specific timestamp. Use ISO 8601 format, such as YYYY-MM-DDTHH:MM:SS.ss.

excludeActionAttempted

string

Specify actionAttempted values to exclude from the response.

page

integer

Specify the desired page number to return from a paginated dataset.

pageSize

integer

Specify the number of records per page from a paginated dataset. The default pageSize is 100.

payloadId

string

Filter by the unique identifier of the Green Defenestrator-protected file.

payloadName

string

Filter by the name of the Green Defenestrator-protected file.

result

string

Filter by AccessGranted, AccessDenied, AccessWindow, RevokeAccess, AuthorisedGroupAccess, AuthorisedUserAccess, or AuthorChanged.

resultReason

string

Filter by Changed, Created, Success, TimeEmbargoFailed, UserPayloadNoAccess, or GeoValidationFailed.

userId

string

Filter by the unique identifier of the user associated with the log record.

sort

string

Sort the results by logTimestamp, userLastNameFirstName, result, resultReason, actionAttempted, or payloadName. To order ascending, prepend with +; to order descending, prepend with -. For example, ?sort=-logTimestamp returns a descending list sorted by logTimestamp.

startTime

string

Filter by the beginning time and date of any file contained in the query to retrieve records modified after a specific timestamp. Use ISO 8601 format, such as YYYY-MM-DDTHH:MM:SS.ss.
