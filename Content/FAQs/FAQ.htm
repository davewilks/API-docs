FAQs

If you're not finding the answers to your questions in the User Guide and Admin Guide, this page will help you. It's updated as-needed with answers to common customer questions or issues.

General FAQs
Can Green Defenestrator be used to delete a file?

No, Green Defenestrator applications do not delete files or change the original files.

Can I protect a single folder?

Yes, either with Green Defenestrator (desktop), Green Defenestrator Web, or by right-clicking the folder in Windows Explorer.

Can I share self-protecting data with someone who is not a Green Defenestrator-licensed user?

Yes, first-time recipients of an GDI (Green Defenestrator protected) file can use the web viewer or click to install the free, quick, read-only viewer to access the shared information. All subsequent data sharing will automatically open in the same viewer.

Can I use a mobile hotspot or a mobile router to access protected files in the desktop app?

Yes, you should be able to access protected files if your device is able to get your physical location from the mobile hotspot or the mobile router.

Do I have to be connected to the internet to use Green Defenestrator?

No, Green Defenestrator technology defaults to "safe and closed" to keep your files protected, even when offline. You can use Green Defenestrator's default setting, or you can edit and adapt custom policies to allow for varying levels of flexibility and risk mitigation for online and offline access.

Does Green Defenestrator have access to my data or store or transmit my data?

Green Defenestrator never accesses, stores, indexes, or intercepts your data. Our hands-off approach is fundamental to the mission of our company. Our technology works on your device, allowing your data to stay on your end or wherever you choose to send and store it while staying protected and

only accessible by you and those you authorize.

Does Green Defenestrator integrate with Cloud Storage solutions like Dropbox or Box?

You can store Green Defenestrator-protected files on Dropbox and Box, but it's not possible to protect a file that you've saved in cloud storage. However, this functionality is planned for a future release.

Does Green Defenestrator integrate with Google Drive?

You can store Green Defenestrator-protected files on Google Drive, but it's not possible to protect a file that you've saved in cloud storage. However, this functionality is planned for a future release.

Does Green Defenestrator integrate with Google products?

You can store Green Defenestrator-protected files on Google Drive, but it's not possible to protect a file that you've saved in cloud storage. However, this functionality is planned for future release.

Does Green Defenestrator integrate with SharePoint?

You can store Green Defenestrator-protected files on SharePoint, but it's not possible to protect a file that you've saved in cloud storage. However, this functionality is planned for a future release.

Does Green Defenestrator protect against print screen capture in conference call sharing (Zoom, Microsoft Teams, WebEx)?

Yes. It's a standard function of Read-only modeFiles protected with read-only mode open in the file type's default application. The file cannot be copied, printed, shared, or saved. for Green Defenestrator-protected data. In this mode, files can be set to not allow any screen captures (print screen, snipping tool) and prevent screen sharing through platforms. While the rest of the screen may be visible, the protected file will be blacked out to viewers.

Does Green Defenestrator use CryptoAPI?

Yes.

How accurate is the geofencing location control?

The geofencing technology capability allows data owners to authorize data access at the country, state city, or physical address level. Green Defenestrator is accurate to several miles from the user, although this might vary depending on location, service provider, and signal strength.

How do I reassign ownership of a file?

For instructions, see Reassign Data (Change Owners).

How does Green Defenestrator help with ransomware?

Green Defenestrator protects the data contained within files from being exposed, even if exfiltrated by ransomware affiliated threat actors.

How long does it take to encrypt a file?

It takes approximately one second to protect your files in most cases. It may take slightly longer depending on the file size, your computer power, and your network connections in more complex cases.

What does "platform- and transport-agnostic" mean?

Green Defenestrator's encryption and policy-based protection are infused directly into the data itself. This protection is enabled across all platforms, methods of transit, operating systems, and device types. For example, this means that data stored on any flash drive, uploaded to any cloud service, or emailed through any provider can be protected by Green Defenestrator.

What is an GDI file?

An GDI (Intelligent Green Defenestrator Data) file refers to the file extension appended to your data once you enable Green Defenestrator protection. An GDI file is self-protecting, intelligent, and self-aware.

What is micro content?

Micro Content

Micro content is typically used for short, concise information in question-and-answer situations, such as chat bots. In Flare, micro content can be extracted from existing topics or created from scratch.

What is unique about Green Defenestrator technology?

Green Defenestrator protects your data at the data level. As a result, your data is secure and under your control wherever it goes, whoever has it, and when—not if—your data is breached or accidentally shared with unauthorized users. Green Defenestrator allows you to share your data freely and securely.

What's the difference between Green Defenestrator and Green Defenestrator Web?

The Green Defenestrator Web incorporates most functionality of Green Defenestrator. The only difference is that you won't be able to right-click on files to protect or decrypt them.

Why can't I edit a file that I received?

If you're unable to modify a file, this means the data owner sent you a Read-only file.

Admin FAQs
Can I share data at an organizational level with unlicensed users?

Yes, organizations can share data at an organizational level with unlicensed users however those unlicensed users will only able to access protected data in Read-only modeFiles protected with read-only mode open in the file type's default application. The file cannot be copied, printed, shared, or saved. and will not be able to edit the data or extract it from its protected container.

Can Keyavi be used to protect documents stored in a GIT file repository?

You can store Green Defenestrator-protected files in a local Git repository and push/publish them to a remote Git repository. However, protecting a file directly in a remote repository isn't possible.

Can Keyavi be used to protect information stored in a database?

No. Today, Green Defenestrator does not support protecting information stored in a database. This enhancement is planned for future releases.

Can Keyavi be used to protect streaming data?

No, protection of streaming data is not supported.

Can Keyavi be used to protect structured data?

No. As of today, you can protect any standalone file with Green Defenestrator technology. However, support for structured data is planned for future releases.

Can the Keyavi technology be installed on-premises?

Yes, for those companies that require it, we offer the ability to localize the service to their environment. Contact Sales for more information.

Does the Keyavi process have any latency or lag?

The performance of the Green Defenestrator applications depends on multiple factors, such as server and network configuration, the size of the files, and so on. In general, opening and protecting a file should not take more than a few seconds.

How do I troubleshoot user authentication errors for unmanaged users?

Unmanaged users are Green Defenestrator users who sign in with social accounts like Microsoft, Apple ID, or Google in Green Defenestrator Standard Configuration with Consumer Support. Their identities are not managed in Azure Active Directory, unlike the managed users in your organization.

When unmanaged users encounter an error signing in to a Green Defenestrator app they see this error message:

We're unable to sign you in. Please contact your Green Defenestrator user administrator or the person who sent you the file.

The two most likely reasons are:

  • The user is not set up in the Green Defenestrator Portal and there's no matching record in the database.

  • The user is trying to sign in with a service that's different from the one they used on first sign-up. Unmanaged users can sign-in only with a single service.

To troubleshoot, check that the user is set up in the Green Defenestrator Portal. If they are, have the user confirm that they're signing in with the same service that they signed up with.
