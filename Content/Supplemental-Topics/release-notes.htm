Keyavi 2.3.0

April 28, 2022

What’s new

Keyavi API

The Keyavi API is now available for integration with customers’ and partners’ tools and services. This first set of available REST APIs provide functionality for user/session authentication and audit and forensics log export. This enables audit and forensics logs to be exported into Security Information & Event Monitoring (SIEM) tools of your choice, such as Azure Sentinel, for timely monitoring and access to detailed information related to the opening, protection, and audit events of Keyavi-enabled files.

Keyavi Mobile Apps

The Keyavi Mobile apps (iOS and Android) are now available in their respective stores. The Keyavi iOS and Android apps enable you to protect files on your iPhone, iPad, and Android devices, send protected files to recipients, open protected files others have given you access to, and manage access to your protected files. Keyavi’s intent is to achieve feature parity across all supported operating systems for seamless user experience. However, due to platform constraints, the Keyavi mobile apps have the following limitations:

  • Keyavi mobile apps support opening protected files only in Read-only and Full Access modes. We don't currently support Edit and Offline modes.

  • Due to iOS platforms limitations, the iOS app cannot restrict screen capture of read-only documents. We recommend that you use Keyavi Desktop applications if you need to prevent screensharing protected content. The iOS app is available in the App Store, and the Android app is available in Google Play. For usage instructions, please see Keyavi Mobile User Guides.

Keyavi Desktop app

  • When a file was protected by Keyavi Desktop app, any previously protected file with the same name in the same directory would be overridden. Now, you will be notified in case a file with the same name already exists so that you can chose whether you want to override that file or not.

Intelligent Directory

  • Intelligent Directory was not able to re-create sub-folder structure in the target directory. Instead, it would put all the processed files in the same root target directory. Now intelligent directory mirrors any sub-folder structure in the target directory.

Bug fixes

  • After uninstalling the Keyavi desktop app, “Decrypt with Keyavi” and “Protect with Keyavi” were still present in the right-click menu options. Now these options are present only if Keyavi desktop app is installed.

  • In some scenarios, Keyavi-licensed read-only users were able to protect files in Keyavi desktop application. Now only Keyavi Full users have access to the protect functionality.

  • In some case (e.g. opening a file from unauthorized location), Keyavi Portal displayed an opening event in a location different from the one it took place in. Now any opening event with enabled location services will have correct geo-information.

Known issues or limitations

  • In some scenarios (e.g. opening a file from a non-authorized location), the message displayed by Keyavi Desktop app provides incorrect information about the reasons of the error. Contact your Keyavi admin or local support.

  • When using multiple environments/profiles in the same browser session, it takes multiple clicks to sort columns in Keyavi Portal / Data Management Console / Chain of Custody and Audit Policy Trail tables. When working in multiple environments, use independent browsers.

  • When using Keyavi Desktop app in out-of-the-box file extension deployment configuration, Microsoft Office files containing some Shape Objects won’t open in Read-only or Edit modes The Authorized User can view and edit the data in a protected view with limited capabilities. Users can make changes, copy, paste, and save the modifications but cannot screen share or save an unprotected copy.. You will need to contact your administrator to enable native file type extension deployment configuration. If this is not possible, replace Shape Objects by images or convert the file to an Adobe PDF extension.

  • In the native file type support deployment, when keeping a large Read-only file opened and idle for over 10 minutes, Keyavi Desktop app will display an error message and no actions could be performed on that file. You need to close the file and open it again to be able to access the content.

  • When using Keyavi Desktop app as a Read-only user, any attempt to protect a file from the Windows right-click menu will cause Keyavi Desktop app to shut down. You will need to re-launch the application to use it again.

Keyavi 2.1.0 macOS

The Keyavi Desktop Application is now available for macOS users. You can now protect and open Keyavi-enabled files on macOS devices, as well as manage Keyavi User Profiles. Here are the features:

  • Read-only mode Files protected with read-only mode open in the file type's default application. The file cannot be copied, printed, shared, or saved. supports Microsoft Office files (except for PowerPoint), text, images, and Adobe PDF. A full list of supported file types is available here.

  • Full Access mode The recipient can view and edit the data in its native application. supports all file extensions

  • Edit mode is not supported for macOS. It will be available in future releases

Known issues or limitations

  • The app login page has a browser check. If the browser isn’t Safari, Edge, Chrome, or a Chromium-based browser, an “unsupported browser” message is displayed.

  • Offline mode is not supported (the icon at bottom is invisible) in Keyavi Desktop app for macOS. You'll need to use Keyavi Desktop for Windows to open files with offline access.

  • Files with Edit access open in Read-only mode in Keyavi Desktop for macOS. You’ll need to use Keyavi Desktop for Windows to open these files in Edit mode.

Keyavi 2.2.0

December 2, 2022

What’s new

This release provides an increased level of cyber security across the entire suite of Keyavi applications, verified by exhaustive static and dynamic security tests, along with software composition analysis, and manual penetration tests. In addition, the new release includes several important bug fixes.

Bug fixes

  • Opening a protected file without being signed into Keyavi was causing Keyavi Desktop to crash. Now any attempt to open protected files without being signed in won’t cause any issues.

  • Opening some files in Read-only mode with native file type support enabled was causing Keyavi Desktop to crash. Now opening files in Read-only mode will work as expected.

  • Renaming protected files would make them inoperable when the original file extension was removed from the file name. Now you should be able to open files with any name.

  • When opening protected files in offline mode with native file extension support enabled, files with Read-only access would open in Full access. Now these files are opened in Read-only mode.

Known issues or limitations

  • The following file types can be opened in Offline Access mode with Read-only access: Microsoft Word (.docx), Microsoft Excel (.xlsx), Text (.txt), Adobe PDF (.pdf), Image (.png, .jpg).

Keyavi 2.1.0

August 15, 2022

What’s new

  • Read-only mode is now available for over 150 file types. You can protect all these file types across all Keyavi products. All these file types can be opened in Keyavi Desktop app. The full list of supported file types is available here.

  • Keyavi now provides comprehensive documentation of logs (user actions and user-triggered events). This lets you create and export audit and forensics reports.

  • External (unlicensed) users can now open standalone protected files using Keyavi Viewer. Previously, it was possible to open files only from a protected email.

  • All Microsoft plugins are deactivated and unsupported. We now provide a seamless experience with the support of native applications.

Bug fixes

Some Keyavi security groups didn’t have the correct scope of rights. That issue has been fixed in the following ways:

  • Read-only users can’t protect files.

  • Only non-admin users can protect and open files.

  • Security groups now have cumulative capabilities.

  • Keyavi Mobile apps were removed from the Apple Store and Play Store. In the future, Mobile applications will be delivered directly to customers.

  • Some users experienced issues when retrieving large number of files in Keyavi Portal. Now, you may retrieve any number of files.

Known issues or limitations

  • In Read-only mode, Keyavi can only open 3-D files up to 250MB in size and thumbnails or image previews if the image size is less than 100 MB (approximately 12000 x 8000 pixels). The full list of impacted file extensions is available here.

  • The user sees an error message when opening a file not supported in Read-only mode (you can still grant Read-only access to any file). The full list of supported file types is available here.

  • When opening certain type file in Read-only mode, clicking on some buttons (e.g. Share) will lead to an unexpected behavior (e.g. a share link pop-up not displaying properly). Usually, it’s possible to go back to the initial screen by pressing the Esc button.

  • Access time violations aren't shown in the Portal/Dashboard. This will be fixed in Keyavi 3.0.

Keyavi 2.0.2

March 28, 2022

What's new

Keyavi desktop application

  • Files protected with Full Access mode are now accessible in both Full Access and in Edit Mode. You can easily modify the contents of a protected without reverting to the native file format.

Bug fixes

  • PDF files were not resizable when opened in Read-only mode. Now you should be able to resize these types of files.

  • Some PDF files were causing Keyavi desktop application to crash. Now you should be able to open any protected PDF.

Known issues or limitations

  • Some users experience issues when upgrading to the latest version. We recommend uninstalling all Keyavi applications and delete “Keyavi Data Corp” folder from C:\Program Files (x86) before installing the new version.

  • Outlook plugin was deactivated. It is not available for installation and any previous versions must not be used.

  • Suspend device feature is not available. The setting is deactivated but still visible in the Portal.

  • Recall status does not update after recalling a file.

  • Sometimes the pop-up with Edit and Full Access options does not disappear when opening a file. User needs to close the pop-up manually by clicking Cancel button.

  • Dragging a protected file from email client (e.g. Outlook) directly into Keyavi desktop application does not open the file. User needs to download the protected file to open it.

  • Any change to user email address in Azure Active Directory disconnects the user from any previously protected files. Keyavi administrator will need to reassign user data to the account with the new email address to restore user access to previously protected files.

Keyavi 2.0.0

January 10, 2022

What's new

General

  • Added enterprise user authentication. From now on, you'll sign into Keyavi applications with your corporate account—no need to create a dedicated Keyavi account or maintain a username and password anymore.

  • Added support to share protected files with groups. You can authorize and manage file access for existing organizational groups of users (created by your admin) in addition to single-user operations.

  • Enhanced sharing capabilities with external recipients. You can now send emails and share files with users that are not part of your organization. External users will register with Keyavi to view received email with attached files in read-only mode using Keyavi Lite or Keyavi Viewer. Simple!

Keyavi desktop application

  • Profile management. You can now add and manage multiple Agent profiles, each conforming to a distinct Keyavi deployment.

Keyavi Portal

  • Functionality to add or remove users has been integrated into Azure Active Directory (AAD). No more manual user management in Keyavi.

Keyavi Viewer

  • Introducing Agent Viewer, a web version of Keyavi Lite for users that cannot download it. Same functionalities as Keyavi Lite.

Enterprise integration

  • Organization administrators can now create groups of users to allow sharing of protected files with groups.

  • Enterprise automated deployment capabilities using YAML.

  • System administrators can now mass deploy Keyavi desktop application using an MSI installation package.

Bug fixes

  • We improved the performance of the Intelligent Directory when processing files.

  • Some PDF and Microsoft documents were not readable in the Web App. Now you should be able to open these types of files.

  • Some Smart Emails were not rendering properly. Now all the recipients should be able to access the files.

  • Occasionally the Keyavi Web link would redirect you to a blank page. Now you will always navigate to the Keyavi Web landing page.

Keyavi Suite of Applications

Keyavi Desktop application (Windows only) - a user-friendly, intuitive Windows desktop application to help you maintain complete control of your protected data. Plugins for the Microsoft Office Suite, including Outlook, Word, Excel, and PowerPoint, allow seamless integration of the Keyavi technology directly into the Microsoft ecosystem.

Keyavi Web (Web browser) - not all users can download and install Keyavi (former name: Agent), so Keyavi offers a web client to protect and open IKD files Short for "Intelligent Keyavi Data" and is the file extension for a Keyavi-protected file. This file has multiple layers of encryption and policy-based rulesets infused into the data itself, making it self-protecting, intelligent, and self-aware.. The Keyavi Web (former name: Web App) incorporates most functionality of Keyavi (former name: Agent). The only difference is that you won't be able to right-click on files to protect or decrypt them.

Keyavi Viewer (Web browser) - a web version of Keyavi Lite (Reader) for users that cannot download the Keyavi Lite (Reader). Same functionalities as Keyavi Lite (former name: Reader).

Keyavi Portal (Web browser) - an administration portal for data owners to monitor and manage their protected data. It's also the backend for the user, policy, and security administrators.

Keyavi Microsoft Office Add-ins - add-ins for the Microsoft Office Suite, including Word, Excel, PowerPoint, and Outlook for user to able to protect Office files natively from the ribbon.

Known issues or limitations

  • Edit mode is not supported for non-office documents as well as PowerPoint files.

  • Because of a browser limitation and its expected behavior, you can take screenshots of read-only files in Keyavi Web and Keyavi Viewer.

  • If connected to a mobile hotspot, for example, tethered to a cell phone or mobile router, Keyavi (desktop) doesn't recognize the location, resulting in offline status. This means only open offline authorized files can be opened.

  • When protecting folders, Intelligent Directory does not mirror the hierarchical structure of nested subfolders but instead saves all the IKDs in a single folder. Therefore, if files in different subfolders share the same name, all but one gets overwritten. Support for mirror directory structure (inclusive of sub-directories) is planned for a future release.

  • Full Access mode doesn’t allow to save changes to the protected copy of the file. This means you will need to access the file in Edit mode to make any changes to the protected copy.
