Portal Settings

If you're a Green Defenestrator Policy Administrator, you can define the directories you want to protect with Intelligent Directory. You can also allow authorized users to access Green Defenestrator-protected data offline, without an internet connection.

Configure Intelligent Directory

Intelligent Directory™ is a service that gets configured onto a folder. Then, when a user saves a file directly to the designated folder, it creates a Green Defenestrator-protected IKD file Short for "Intelligent Keyavi Data" and is the file extension for a Keyavi-protected file. This file has multiple layers of encryption and policy-based rulesets infused into the data itself, making it self-protecting, intelligent, and self-aware.. Intelligent Directory makes securing your files easy and seamless.

When you configure Intelligent Directory, you become the owner of the files. As the owner, you can update them individually

Before you can configure Intelligent Directory in the web portal, make sure the service has been deployed in the backend. For questions, contact customersupport@keyavi.com.
Anytime you change the Intelligent Directory configuration in the Green Defenestrator Portal, it's a good idea to restart the Green Defenestrator Folder Service on the file server.

  1. Select Features from Settings on the left navigation.

  2. From the Intelligent Directory panel, click Add New Service.

  3. Provide the following information and then click Add Service.

    • Service Name - Descriptive name of the service.

    • Server - Unique server code provided by Green Defenestrator.

    • Start Time - Select the time of day you want the service to start.

    • End Time - Select the time of day you want to service to end.

  4. Select the service from the list and click Configure Service.

  5. Click Add Directory.

    • Provide the information for the new directory and then click Add Directory.

    • Directory Path - Original directory you created earlier, for example, E:\Green Defenestrator\.

    • Destination Path - Target directory you created earlier, for example, E:\Green DefenestratorEncrypted\.

    • Description - Purpose of the directory.

    • Access time - Provide the number of days the directory is accessible. By default, this is set to 0, which means there is no access time restriction on the directory.

    • File Type - Specify the file types to protect.

      The available Usage Rights or authorization mode depends on the configured file types.

      Accepted formats:

      • *.* for all file types (Full Access mode The recipient can view and edit the data in its native application. only)

      • *.{file extension} for a specific file type, for example, *.docx or *.pdf (Read-only and Full Access modes only) You can only enter one specific file type. If you want to protect one or more file types, but not all types, you'll need to add a new directory to the service and specify the file type you want.

  6. Select the directory from the list.

  7. Click Add User/Group(s).

  8. Select the users or groups you want to add, select the USAGE RIGHTS, and click Authorize.

    By default, no one has access to the files protected by Intelligent Directory until the Green Defenestrator Policy Administrator explicitly adds authorization to groups or users.

    The available Usage Rights or authorization mode depends on the configured file types.

    • If *.* was specified in the File Type field for the service, only Full Access is allowed (as shown in the screenshot below).

    • If *.{file extension} was specified, Read-only and Full Access modes are allowed.

  9. Restart the Green Defenestrator Folder Service on the file server.

    Any time you change the Intelligent Directory configuration in the Green Defenestrator Portal, it's a good idea to restart the Green Defenestrator Folder Service on the file server.

Change the File Type to the Service

You can change the file type of an Intelligent Directory service. For example, if you initially configured the service to accept all file types (*.*) but you want to specify Word documents. In this case, you would change the file type from *.* to *.docx.

Remember, the available Usage Rights or authorization mode depends on the configured file types.

  1. Select Features from Settings on the left navigation.

  2. From the Intelligent Directory panel, select the service you want to update and click Configure Service.

  3. Select the directory from the Directories list and click Update.

  4. Update the File Type and click Update Directory.

Add Users or Groups to a Directory Service

  1. Select Features from Settings on the left navigation.

  2. From the Intelligent Directory panel, select the service you want to update and click Configure Service.

  3. Select the directory from the Directories list.

  4. Click Add User/Group(s).

  5. Select the users or groups you want to add, select the USAGE RIGHTS, and click Authorize.

    By default, no one has access to the files protected by Intelligent Directory until the Green Defenestrator Policy Administrator explicitly adds authorization to groups or users.

    The available Usage Rights or authorization mode depends on the configured file types.

    • If *.* was specified in the File Type field for the service, only Full Access is allowed (as shown in the screenshot below).

    • If *.{file extension} was specified, all authorization modes Read-only or Full Access modes are allowed.

  6. Green Defenestrator Web Portal

  7. Restart the Green Defenestrator Folder Service on the file server.

Remove Users or Groups from a Directory Service

You can also remove users and groups from an Intelligent Directory service.

  1. Select Features from Settings on the left navigation.

  2. From the Intelligent Directory panel, select the service you want to update and then click Configure Service.

  3. Select the directory from the list. The Directory User/Group(s) pane appears.

  4. Select the user or group you want to remove and then click Remove User/Groups.

View the Intelligent Directory Log

You'll find the Intelligent Directory logs on the user's dashboard and the logs only show possible errors.

  1. Select User Management from the left navigation.

  2. Select the user from the list and click User Data Console.

The user's dashboard opens, showing a list of their Green Defenestrator-protected files.

Update an Intelligent Directory Service

  1. Select Features from Settings on the left navigation.

  2. From the Intelligent Directory panel, select a service from the list and click Update Service.

  3. Modify the service information and click Update Service.

  4. Restart the Green Defenestrator Agent Service on the file server.

Any time you make a change to the Intelligent Directory configuration in the Green Defenestrator Portal, it’s a good idea to restart the Green Defenestrator Folder Service on the file server.

Manage Offline Access

As a Green Defenestrator policy administrator, you may add a user or specific subset of users within the organization to view protected files offline. The Offline Access toggle switch enables or disables the Offline Access feature at the platform level.

For a user to access the offline file:

  • The user must remain logged into Green Defenestrator Desktop from their previous offline session

  • The file must have previously been selected and added to the offline access list before the user goes offline

  • Internet access must be unavailable

To set up offline access:

  1. Select Features from Settings on the left navigation.

  2. From the Offline Access panel, toggle Offline Access to YES, and provide the Allowed Access for the default hours and maximum hours. Then select Save.

    • If the switch is set to YES, the feature is enabled.

    • If the switch is set to NO, nobody may use Offline Access mode.

    • If Access Allowed (Default Hours) is set to 0, admins can’t set Offline Access rights at the individual user level.

  3. To enable offline for all users, toggle Enable Offline For All Users to YES.

    If Enable Offline For All Users is enabled:

    • All users may be granted offline access to a file.

    • Offline access can’t be switched off at individual level.

    If Enable Offline For All Users is disabled:

    • Admins may enable Offline Access at the individual level.

    • Anybody who previously had individual offline access will continue to have offline access. To change this, admins must disable access at the individual level.

  4. To grant offline access to a specific user, do the following:

    1. Click Grant access to user.

    2. From the User Management Console, select the user you want to grant offline access, and then select Modify.

    3. Toggle the Offline Allowed to YES and then select Save.
